# Security Policy

## Supported Versions

Until Shiori has stable release branches, security fixes are made against the latest code on the default branch.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue with exploit details.

At the moment, this repository does not publish a dedicated private disclosure channel in-tree.
If the hosting platform provides private vulnerability reporting for the repository, use that.
Otherwise, open a minimal public issue that asks for a secure contact route without including exploit details, payloads, or reproduction steps.

Include:

- a description of the issue
- reproduction steps
- impact assessment
- any suggested remediation

We will acknowledge reports as quickly as possible, work on a fix, and coordinate disclosure timing before public release of details when appropriate.

Please keep proof-of-concept code, payloads, private data, and reproduction steps out of public issues until a fix is available.
